# Xinbot 使用指南

[English](USAGE.md) / 简体中文

本指南介绍了 Xinbot 的核心概念，并提供了内置命令的完整参考。

---

## 1. 核心概念

### Bot (机器人)
连接到 Minecraft 服务器的核心客户端。它负责处理网络会话、身份验证（包括微软正版账号）并协调所有其他系统。

### Plugin (插件)
用于为机器人添加自定义逻辑的可扩展模块。Xinbot 采用插件优先的架构，允许您在不重启程序的情况下加载、卸载或重载功能。

### Event (事件)
机器人内部触发的信号（例如：收到聊天消息、登录成功）。插件可以“监听”这些事件并自动做出响应。

### Packet (数据包)
机器人与服务器之间交换的底层网络数据单元。高级插件可以拦截原始数据包以实现精细控制。

### Command (命令)
用于手动控制机器人行为的控制台指令。命令支持 Tab 补全和通过 JLine 实现的实时语法高亮。

---

## 2. 命令参考

Xinbot 命令可以直接在控制台中执行。命令及其参数不区分大小写。

| 命令 | 别名 | 用法 | 描述 |
| :--- | :--- | :--- | :--- |
| `help` | - | `help [命令]` | 显示所有可用命令或特定命令的帮助信息。 |
| `say` | `chat` | `say <消息>` | 向服务器公共聊天频道发送消息。 |
| `command` | `cmd` | `cmd <指令>` | 向服务器发送指令（例如 `cmd home` 会发送 `/home`）。 |
| `pm` | `PluginManager`| `pm <子命令>` | 管理插件（列表、加载、卸载、重载等）。 |
| `plugins` | - | `plugins` | 列出当前所有已加载的插件及其版本。 |
| `list` | - | `list [uuid]` | 列出在线玩家。使用 `uuid` 参数可查看玩家 ID。 |
| `disconnect` | - | `disconnect` | 从当前服务器断开连接。 |
| `stop` | - | `stop` | 停止机器人并安全关闭程序。 |
| `license` | - | `license` | 显示 GPL-3.0 许可证信息。 |

### PluginManager (pm) 子命令
- `pm list`: 列出所有插件。
- `pm load <文件名>`: 从插件目录加载指定的 JAR 插件。
- `pm unload <插件名>`: 卸载插件。
- `pm reload <插件名>`: 重载插件。
- `pm enable/disable <插件名>`: 启用或禁用插件。
- `pm re-enable <插件名>`: 重启（禁用并立即启用）插件。

## 3. 进阶功能

### Tab 补全
Xinbot 拥有强大的基于 JLine 的控制台。按下 `Tab` 键可以补全命令名、子命令、插件名，甚至在使用 `cmd` 时补全服务器侧的指令。

### 语法高亮
控制台中的命令会实时高亮。有效的命令和参数会显示为特定颜色（如青色、蓝色），而无法识别的输入则显示为红色，有助于避免拼写错误。

### 服务器指令
若要向 Minecraft 服务器发送指令（如 `/w`），**请勿**直接输入，而应使用 `cmd` (或 `command`) 前缀：
> `cmd w <用户名> <消息>`

---

## 4. 使用技巧

- **管理员设置**: 确保 `config.conf` 中的 `owner` 字段与您的 Minecraft 用户名一致，以便插件识别您的管理员身份。
- **自动斜杠**: 使用 `cmd` 发送指令时，控制台会自动处理开头的 `/` 符号。
- **退出程序**: 建议使用 `stop` 命令退出，以确保所有插件在程序关闭前都能被正确卸载。
